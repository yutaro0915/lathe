同一コマンドパターンの反復失敗ループ（gh issue view --comments）

現象

「Swiftで新規プロジェクトを作成」および「ML性能をレビュー」の複数セッションにわたり、gh issue view # --comments コマンドが最大15回の反復非ゼロ終了を記録した。リポジトリ指定の有無を問わず同パターンの失敗ループが複数コンテキストで再現しており、セッション横断的な現象である。

Analysis analyst の構造化推論
impact

重要度高 (conf 95%): エラー文が原因（Projects classic 廃止）を明示しているのに手法を変えず20回反復し、トークン・時間を空振りに費やした。最終的に seq 2421 で --json number,title,body,comments に切り替えて回避できており、「エラー本文を読んで手法を変える」という1ステップが欠けていた。放置すると gh を使う全 close 判断・issue 監査でこのループが再発する。

agent_intent

issue 監査サブエージェントが、close 可否を判断するために各 issue の本文とコメントを読み取ろうとしていた（直前 turn: 「Python 側の単体テストは15件通過。次に差分を見て計測 CSV と perf report の変更が issue に反映されているか詰める」）。手段として gh issue view N --comments を選び、issue 2/3/6/7 に対し繰り返した。

cause_hypothesis

仮説: gh CLI が issue 本文取得時に内部で Projects (classic) GraphQL フィールド (repository.issue.projectCards) を必ず叩き、これが sunset 済みのため gh issue view N --comments が exit 1 を返している。agent はエラー本文が「Projects 廃止」を指していることを手掛かりに使わず、コメント取得という目的を達するため番号だけ変えて同じ --comments 形を15〜20回反復した。
